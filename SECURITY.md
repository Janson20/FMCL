# 安全策略

## 报告漏洞

如果您发现 FMCL 存在安全漏洞，**请勿**在公开的 Issue 中披露。
请通过以下方式私下报告：

1. **创建安全 Issue**：在 GitHub 仓库中创建一个 Issue，标题加 `[SECURITY]` 前缀，并在描述中说明这是安全相关问题。
2. **直接联系维护者**：通过仓库主页显示的邮箱联系项目维护者。

### 报告内容应包含

- 漏洞的详细描述
- 复现步骤（PoC 优先）
- 受影响的版本
- 可能的缓解方案（如有）

## 响应时间

- 我们会在 **48 小时内**确认收到报告
- 严重漏洞力争在 **7 天内**发布修复版本
- 修复完成前不会公开漏洞细节

## 安全最佳实践

- 始终从官方 GitHub Release 下载安装包
- 不要运行来源不明的第三方插件（`.fmpl` 文件）
- 启动器加密密钥文件（`.fmcl_key`）请妥善保管，丢失将无法解密已存储的凭据
- 定期更新到最新版本以获取安全修复
